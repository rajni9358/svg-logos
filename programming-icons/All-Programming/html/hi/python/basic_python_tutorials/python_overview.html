Python का परिचय

Python एक उच्च-स्तरीय, इंटरप्रेटेड, इंटरैक्टिव और ऑब्जेक्ट-ओरिएंटेड स्क्रिप्टिंग भाषा है। यह अन्य भाषाओं की तुलना में अधिक पढ़ने योग्य है और इसमें कम सिंटैक्स की जटिलताएँ हैं।

  • Python इंटरप्रेटेड है – रनटाइम पर इंटरप्रेटर द्वारा प्रोसेस की जाती है।
  • Python इंटरैक्टिव है – आप इंटरप्रेटर से सीधे बातचीत कर सकते हैं।
  • Python ऑब्जेक्ट-ओरिएंटेड है – ऑब्जेक्ट ओरिएंटेड प्रोग्रामिंग को सपोर्ट करता है।
  • Python शुरुआती लोगों के लिए है – नए प्रोग्रामर के लिए बेहतरीन विकल्प।

यह एक ओपन-सोर्स और क्रॉस-प्लेटफॉर्म प्रोग्रामिंग भाषा है, जो सभी प्रमुख ऑपरेटिंग सिस्टम पर उपलब्ध है।

नई सुविधाओं के लिए Python Enhancement Proposal (PEP) प्रक्रिया मौजूद है, जिसमें कोई भी नई विशेषता या सुधार प्रस्तावित कर सकता है।

Python की डिज़ाइन फिलॉसफी सादगी, स्पष्टता और पठनीयता पर केंद्रित है। इसे "बैटरियाँ शामिल हैं" दृष्टिकोण के लिए जाना जाता है।

Zen of Python में इसके दर्शन को निम्नलिखित aphorisms के रूप में बताया गया है:

  • सुंदर बदसूरत से बेहतर है
  • स्पष्ट अप्रत्यक्ष से बेहतर है
  • सरल जटिल से बेहतर है
  • जटिल जटिलतापूर्ण से बेहतर है

पूरा Zen प्राप्त करने के लिए, Python शेल में टाइप करें:

>>> import this

यह निम्नलिखित 19 सूक्तियाँ देगा:

सुंदर बदसूरत से बेहतर है।
स्पष्ट अप्रत्यक्ष से बेहतर है।
सरल जटिल से बेहतर है।
जटिल जटिलतापूर्ण से बेहतर है।
सपाट नेस्टेड से बेहतर है।
फैले हुए घने से बेहतर हैं।
पठनीयता मायने रखती है।
विशेष मामले नियमों को तोड़ने के लिए काफी खास नहीं हैं।
हालाँकि व्यावहारिकता शुद्धता पर भारी पड़ती है।
त्रुटियों को चुपचाप नहीं गुजरना चाहिए।
जब तक कि उन्हें स्पष्ट रूप से चुप नहीं कराया गया हो।
अस्पष्टता की स्थिति में, अनुमान लगाने के प्रलोभन से बचें।
एक – और संभवतः केवल एक – स्पष्ट तरीका होना चाहिए।
हालाँकि वह तरीका पहले स्पष्ट न हो जब तक कि आप डच न हों।
अब कभी नहीं से बेहतर है।
हालाँकि कभी-कभी *अभी* से बेहतर होता है।
यदि कार्यान्वयन को समझाना कठिन है, तो यह एक बुरा विचार है।
यदि कार्यान्वयन को समझाना आसान है, तो यह एक अच्छा विचार हो सकता है।
